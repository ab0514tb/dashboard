strictモード

"use strict";
をファイルの先頭につけることで、strictモードで動作することができる。 strictモードでは、Javascriptで従来受け入れていた一部のミスをエラーにすることができる。 例えば以下のようなものがある。

偶発的なグローバル変数作成禁止

      var variable1 = 1; // OK
      variable2 = 2; // NG

暗黙的に失敗する代入の禁止

      NaN = 'nan'; // NG

削除できないプロパティの削除禁止

      delete Object.prototype; // NG

関数の引数名などの重複禁止

      function sum(param1, param1, param3) {...} // NG

8進数の禁止

      var x = 0123; // NG

withの禁止

      with (document) {...} // NG

evalの制限

      eval("var x; delete x;"); // NG

一部予約語への代入禁止

      eval = 1; // NG
      arguments = 2; // NG
      implements = 3; // NG
      interface = 4; // NG
      let = 5; // NG
      package = 6; // NG
      private = 7; // NG
      protected = 8; // NG
      public = 9; // NG
      ...
